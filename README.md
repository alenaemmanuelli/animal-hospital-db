# Animal Hospital Database System

## Overview
This project is a relational database created for a mock animal hospital. The goal was to design a system that can store and organize data such as pet information, owners, appointments, and medical records.

This project focuses on database design rather than building a full application.

## What I Did
- Designed a relational database using MySQL
- Created tables for pets, owners, appointments, and medical records
- Set up relationships between entities (ex: owners → pets, pets → appointments)
- Wrote basic SQL queries to retrieve and manage data

## Features
- Store pet and owner information
- Track appointments and visit details
- Record basic medical history
- Support different user roles (receptionist, vet, etc.)

## Example Queries
Some example queries included:
- Viewing all appointments for a pet
- Retrieving a pet’s medical history
- Looking up owner contact information

## Tools Used
- MySQL
- SQL

## What I Learned
- How relational databases are structured
- How tables connect using primary and foreign keys
- Writing basic SQL queries for real use cases
- Thinking about how real systems organize data
